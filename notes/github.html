Git Commands

  • git diff: To see modified file difference with git committed file
  • git add file_name: Add file to the staged area before commit
  • git commit -m "message": Commit the new changes in file (-m means message)
  • git log: See the latest commit log
  • git log --oneline: See all the commits in a concise format
  • git branch: See the current branch in your repository
  • git branch branch_name: Create a new branch (e.g., fix/ or feat/)
  • git checkout branch_name: Switch to another branch (* denotes the current branch)
  • git checkout -b branch_name: Create and switch to the new branch
  • git merge branch_name: Merge a branch into the current branch
  • git branch -d branch_name: Delete a branch (-d stands for delete)
  • git rebase branch_name: Rebase a branch instead of merging
  • git rebase --continue: Continue after resolving conflicts
  • git stash: Temporarily store changes aside
  • git stash list: View the stashed changes
  • git stash pop: Bring the changes back to the current branch
  • git stash show: View a condensed version of the latest stash
  • git stash show -p: View detailed changes in the latest stash (-p is for patch)
  • git stash apply: Add stash list to your working tree
  • git stash show stash@{1}: View a specific stash by name (e.g., stash@{1})
  • git stash show -p stash@{1}: View detailed changes in a specific stash
  • git stash drop stash_name: Drop a specific stash item
  • git reset: Travel to any point in your commit history
  • git reset HEAD~1: Reset to the previous commit
  • git reset HEAD~1 --soft: Keep changes in the working tree
  • git reset HEAD~1 --hard: Discard changes from the working tree
  • git revert HEAD~1: Revert the most recent commit

Note: The code from the stash was added to your working tree, and the stash is still there in case you want to add it somewhere else. Stash the changes again if needed.
